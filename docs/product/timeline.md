## Risk & Buffer

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| LLM output quality requires many iterations | High | Medium | Add 1-day buffer per agent sprint |
| Phaser.js virtual office complexity underestimated | Medium | High | Use plain CSS/React office in MVP 4 if blocked |
| Open questions (OQ-001–OQ-005) unresolved before Sprint 6 | Medium | High | Resolve all Critical OQs before Sprint 6 starts |
| Sprint 11 (Architect Agent) scope creep | High | Medium | Limit to Architecture + DB + API Spec only; defer Security and Deployment design |
| PostgreSQL + pgvector setup issues | Low | Medium | Document exact Docker setup in infra/ early |

## Definition of Done Per Sprint

A sprint is complete only when:

- [ ] All required files are created and committed
- [ ] Every output links to at least one requirement ID (FR-XXX or NFR-XXX)
- [ ] `PROGRESS.md` is updated with sprint status
- [ ] Git commit is pushed with message format `feat(sprint-N): <description>`
- [ ] If backend/frontend code: app starts without errors, Swagger UI works (backend)
- [ ] If agent implementation: agent produces valid JSON output for a sample input
